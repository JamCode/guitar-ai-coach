练习 Tab：页面与点击去向

依据 swift_ios_host 中「练习」Tab（PracticeHomeView / PracticeLandingView.swift）与 swift_app 包内视唱练耳视图梳理。下图箭头表示用户操作后的界面跳转：实线为导航栈 push，虚线为 Sheet，点线为系统 Alert。 首次进入练习 Tab 时先显示占位页，切换一次后即挂载真实首页（见 SwiftEarHostApp.swift）。

实线：导航栈 push（NavigationLink） 虚线：Sheet 弹出层或 Alert（图中统一用虚线表示非 push 交互）

各页交互要点（与上图对应）

练习首页

PracticeLandingView.swift · PracticeLandingView
  • 下拉：刷新本地练习记录
  • 训练日历 → 训练日历页
  • 视唱练耳 → 视唱练耳 Hub
  • 练琴 → 练琴 Hub

训练日历

PracticeLandingView.swift · PracticeCalendarScreen
  • 图形化 DatePicker、当日记录列表（无二级页）

视唱练耳 Hub

PracticeLandingView.swift · EarPracticeHubScreen
  • 四个卡片分别进入音程 / 和弦听辨 / 和弦进行 / 视唱设置

音程识别

swift_app · IntervalEarView.swift
  • 题内：播放、四选一、提交、下一题 / 查看结果
  • Alert「本轮完成」· 确定 → 弹出栈回到 Hub

和弦听辨 / 和弦进行

swift_app · EarMcqSessionView.swift
  • 标题分别为传入的 title（和弦听辨 / 和弦进行）
  • Alert「本轮完成」· 确定 → 回到 Hub

视唱训练 · 设置

swift_app · SightSingingViews.swift · SightSingingSetupView
  • 开始训练 → 视唱训练会话

视唱训练 · 会话

swift_app · SightSingingSessionView
  • 判定、下一题 / 查看结果 → Alert 训练完成 → 确定回到 Hub

练琴 Hub

PracticeLandingView.swift · GuitarPracticeHubScreen
  • 内置三项：和弦切换、节奏扫弦、音阶爬格子（kDefaultPracticeTasks）

和弦切换 · 配置

ChordPracticeSelectionView.swift
  • 和弦进行 行 → Sheet 选择和弦进行
  • 开始练习 → 和弦练习会话（底部主按钮）

选择和弦进行 · Sheet

ChordPracticeSelectionView.swift · ChordProgressionPickerSheet
  • 点列表项选中并关闭 Sheet；关闭取消

和弦切换 · 练习会话

ChordPracticeSessionView.swift
  • 点和弦名 → 指法速查 Sheet（多把位 TabView）
  • 开始 / 暂停 / 结束 → 结束打开保存 Sheet
  • 返回 chevron → 离开确认 Alert
  • 保存成功 → 记录已保存 → 返回 → 回到配置页

节奏扫弦

RhythmStrummingView.swift
  • 完成练习 → 保存练习 Sheet
  • toolbar ? → 图示说明 Alert
  • 返回 → 离开确认（文案与和弦略有不同）

音阶爬格子 · 计时

PracticeTimerSessionView.swift
  • 开始 / 暂停 / 结束 → 保存 Sheet（与和弦会话同类对话框）
  • 返回 → 离开确认；需先计时再结束，否则「先开始练习再结束」

保存练习 · Sheet

PracticeFinishDialog.swift
  • 取消关闭 Sheet；保存记录 → 写本地后关闭并触发后续 Alert（依父视图逻辑）
