# 视唱训练 · UI/UX 备忘（分支 `cursor/6c75954f`）

## 1. 可视化主方向（2026-04-19）

**结论（产品）：** 不以「横轴 = 时间、纵轴 = 相对音分偏差」的**折线/曲线**作为主反馈；改为以 **音高柱状图** 为主：**示范（目标）音高** 与 **用户演唱音高** 并排或对齐展示，形成直观高低对比。

**「稳 / 抖」的表达：** 不依赖时间轴上的曲线形态；通过 **音柱高度随拾音采样更新** 时的**连续小幅上下变化**（或短窗平滑后的缓慢漂移）让用户感知稳定度；必要时可用极轻的透明度脉动表示「正在拾音」，避免喧宾夺主。

## 2. 题型与柱布局（低保真级）

| 题型 | 建议柱布局 | 说明 |
|------|------------|------|
| 单音 | **目标柱**（固定，来自题面） + **用户柱**（实时或判定窗内聚合高度） | 纵轴须标 **音名或 MIDI 刻度**，避免只有相对高度 |
| 音程 | **目标低 / 目标高** 两柱固定 + **用户**在分段判定时对应更新（先低后高）或四柱并排 | 与「先唱低音再唱高音」的文案一致 |

## 3. 与现有实现的关系

- 当前代码：`SightSingingPitchGraphView`（Canvas 时间窗曲线）+ `livePitchCents` 等。  
- **演进策略（建议）：**  
  1. **拾音可靠性**（`sight-singing-evaluate-capture-design.md` 中 P0–P1）可先落地，柱状图**复用同一套** `currentHz` / 聚合 MIDI。  
  2. 柱状图 **MVP** 上屏后，再将曲线 **降级**为次要（折叠、「高级」）或 **移除**，避免双轨信息过载。  
  3. 无障碍：除颜色外须有 **标签/高度差文字**（如「+12¢」）可选展示。

## 4. 与 `sight-singing-evaluate-capture-design.md` 的关系

- 该 spec 的 **§8 非目标** 中「整页动画与曲线视觉语言」指 **不单独做大动效工程**；**以音柱替代主时间线** 属于本文件所述的 **信息架构调整**，可与拾音分期 **并行或紧随其后**，并在 `implementation-plan-sight-singing-evaluate-capture.md` 中另起 **Task UI-Bar**（待排期）。
