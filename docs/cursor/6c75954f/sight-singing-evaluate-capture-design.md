# 视唱训练 · 判定阶段拾音可靠性 — 设计说明

> 归档分支：`cursor/6c75954f`  
> 日期：2026-04-19（修订：示范后直通判定；可视化见 `ui-ux.md`；**底栏：示范与下一题并行、无跳过示范**，以 `ui-ux.md` §6 为准）  
> 来源：头脑风暴已确认范围（优先 A：判定阶段「像没采到 / 分数乱飘」，真机麦克风，与示范/第几次判定无明显规律）

## 1. 背景与问题陈述

视唱训练中，用户在**真机、本机麦克风**下反馈：点「开始判定」后的数秒内，**经常出现极低分或结果像未拾音**，且与是否刚播「示范」、是否同一题第二次判定**无稳定规律**。  
这与实现上「判定窗口内仅统计 `currentHz != nil` 的离散样本」高度一致：若检测器在该窗口内大量输出 `silent` / `rejected`，有效样本过少，则分数与「偏差」对用户不可信。

## 2. 当前实现摘要（代码事实）

- **检测链路**：`DefaultSightSingingPitchTracker` → `TunerPitchDetector(config: .sightSinging)`（与调音器模块同源能力，配置键名为视唱场景）。
- **判定循环**（`SightSingingSessionViewModel.evaluate()`）：对每个目标音，在 `warmupMs + evalMs` 内按 `sampleStepMs`（120ms）节拍唤醒；仅当 `elapsed > warmupMs` 且 `currentHz` 非空时，计算 `abs(midi差)×100` 并入 `absCents`；再交给 `computeSightSingingScore`。
- **空样本退化**：`computeSightSingingScore` 在空数组时返回 0 分等退化结果，用户侧易被理解为「没采到还打分」。

## 3. 目标与成功标准

### 3.1 目标

在**不重做整页视觉大改版**的前提下：

1. 提升判定阶段**有效基频样本的覆盖率与稳定性**，并避免「样本极少却仍给出看似正式的低分」的误导。  
2. **消除「听完示范还要再点一次才开始录/判定」的割裂感**：示范音频（单音或音程整段）**播放结束后**，用户应能**直接开唱**并由系统进入与当前 `evaluate()` 等价的拾音与打分流程，**无需再单独点「开始判定」**。底栏上 **「示范」与「下一题」并列**（见 `ui-ux.md` §6），**不提供「跳过示范直接判定」**。

### 3.2 成功标准（可随日志标定后微调数值）

1. **样本量**：在内部日志中，典型稳定长音场景下单次判定窗口内 **有效样本数 ≥ 某下限**（具体阈值在实现前用带指标包跑 3～5 台机后写入实现计划）。  
2. **稳定性**：同一环境、近似同强度哼唱同一目标，**多次判定的分数方差** 明显低于当前版本（对比同一 commit 前后日志）。  
3. **产品行为**：若窗口内有效样本 **低于约定下限**，**不向用户展示「极低分即结论」**；改为明确提示「未稳定拾音，请重试」或等价文案，并可选择不调用 `submitAnswer` 或提交带「无效」标记的答案（由实现计划与后端约定二选一，默认优先不误导用户）。  
4. **直通流程**：用户在一次「示范」播完后，**零额外点击**即可进入拾音判定窗口（允许极短收尾间隔，如 200～400ms，避免与扬声器串音，具体数值在实现计划中根据真机标定）；手测清单中须包含「播完立即唱」与「停半拍再唱」两条。

### 3.3 交互需求：示范结束后直通拾音与判定

**动机**：当前为「示范」与「开始判定」两步，听感与动作不连贯，用户期望**听完就能唱、唱完即出分**（或进入与现有一致的判定窗口）。

**行为定义（须实现）**

- 在用户触发**示范**（`playPreview` 成功启动）且**整段示范正常播完**后，系统自动进入**拾音判定**（与当前 `evaluate()` 相同的采样、打分与提交语义），**不要求用户再次点击「开始判定」**。  
- **单音题**：示范结束 → 极短间隔（防串音）→ 进入判定（含既有 `warmupMs` + `evalMs` 语义，若与「防串音间隔」叠加冲突，以实现计划为准并更新日志字段说明）。  
- **音程题**：以**整段示范**（低音 → 高音）播放完毕为「可开唱」起点；其后判定逻辑仍须覆盖两个目标音的既有分段（顺序与现版一致）。  
- **不设「跳过示范」**：不想听示范的用户可 **「下一题」** 换题；若需判定须先点 **「示范」** 走完整示范再自动判定（产品取舍，见 `ui-ux.md` §6）。

**边界与防误触**

- 示范**进行中**：判定不得启动；若用户再次点「示范」，以实现计划约定为准（**建议**：取消当前武装，重新播完后再次直通判定）。  
- **判定进行中 / 加载中**：禁用示范或排队策略在实现计划中写清，避免并发两次 `evaluate`。  
- 扬声器播放结束瞬间麦克风可能拾到残响：依赖 3.3 中的**短收尾间隔** + 既有 `warmupMs`；若日志显示串音仍严重，在 P3 或专项中考虑更长的仅间隔、或降低预览音量（非本期必做，仅作备注）。

**界面层（允许小改，不算整页重做）**

- 底栏为 **「示范」与「下一题」并列**（`ui-ux.md` §6）；示范即「播完自动判」路径；**须**在实现计划中给出禁用态与进行中文案。  
- 实时曲线与「判定中」提示应在直通触发时与手动点「开始判定」时**表现一致**，避免用户误解系统未响应。

## 4. 根因假设（待量化验证）

| 编号 | 假设 | 验证方式 |
|------|------|----------|
| H1 | `TunerPitchDetector(.sightSinging)` 在短窗内拒帧比例过高 | 记录每题 `nil` 与有效帧计数、可选 RMS |
| H2 | 单点读取 `currentHz` 对瞬时 dropout 敏感，需时间聚合 | 对比「单点」与「40～80ms 中位数/多数票」样本曲线 |
| H3 | `graphTickMs` 后台任务与判定循环争用 MainActor / 读同一 tracker 导致偶发读空 | 试验：判定期间暂停监控任务 vs 对照 |
| H4 | 用户开唱相对 `warmupMs` 偏晚，有效段被压缩 | 日志中标注首个有效样本时间戳 |

以上不互斥；**第一期以 H1+H2 为主**，H3 做低成本 A/B，H4 用日志确认是否需要动态热身。

## 5. 方案选项与取舍

### 5.1 路线 1：仅调参

只调 `sightSinging` 检测器阈值与 `warmupMs` / `evalMs`。  
**优点**：改动面小。**缺点**：无指标时易盲调；若根因在采样策略，收益有限。

### 5.2 路线 2：采样与抗抖（推荐核心）

- 在判定循环内维护**短环形缓冲**，对每个 `sampleStepMs` 节拍输出**聚合后的频率或 MIDI**（如中位数、trimmed mean）。  
- **有效样本下限**：低于阈值则走「提示重试」分支，避免伪分数。

### 5.3 路线 3：并发与节奏（推荐试验）

- 判定进行中**暂停** `startPitchMonitoringIfNeeded` 的定时追加（或降低与判定读同一状态的冲突）；判定结束恢复。  
- 若 profiling 证明无争用，可回滚该支路。

**综合推荐**：**路线 2 为必做**，**路线 1 在路线 2 落地后**用日志驱动微调，**路线 3 做开关化试验**。

## 6. 交付分期

| 阶段 | 内容 | 验收 |
|------|------|------|
| P0 | 判定过程结构化日志（每题：窗口长、有效样本数、首有效时间、可选拒因枚举） | 真机复现一次即可在控制台或本地文件看到完整字段 |
| P-UX | **示范结束自动衔接 `evaluate()`**（含防串音短间隔、与 `playPreview`/加载态互斥）；底栏 **示范 ∥ 下一题** 按 `ui-ux.md` §6 | 手测：点示范播完即进入拾音；下一题可换题；无「跳过示范」入口 |
| P1 | 环形缓冲 + 聚合读数 + 有效样本下限与 UI 提示 | 样本不足时不出现「极低分结论」；稳定哼唱样本数达标 |
| P2 | 可选：判定时暂停曲线任务；对比 H3 | 若指标无增益则默认关闭 |
| P3 | 视日志结果微调 `TunerPitchDetector` 视唱配置与 `warmupMs` | 成功标准 2 达标 |

**分期说明**：`P-UX` 与 `P0` 可同迭代并行；若自动衔接会改变日志中「判定起点」时间戳，须在 P0 字段中区分 `evaluateTrigger: manual \| postPreview`。

## 7. 测试计划

- **设备**：至少 1 台真机；若条件允许，加 1 台较旧机型。  
- **场景**：安静室内；稳定长音；小声与正常音量各一组；音程题与单音题各若干；**外加**：仅示范直通、**下一题换题**、示范中途再次点示范、播完立即唱与停半拍再唱。  
- **记录**：每场景保存脱敏日志（无原始音频），对比改动前后有效样本数与分数方差。  
- **自动化**：在 macOS 上可保留/补充 `computeSightSingingScore` 单元测试；**拾音路径**以真机手测清单为主（P0 日志可辅助 CI 仅编译开关）。

## 8. 非目标（本期不做）

- **大动效工程**（整页转场、复杂粒子等）仍不做；**主视觉从时间–音分曲线迁到音高柱状对比** 的产品结论写在 **`ui-ux.md`**，实现上可与拾音分期并行或紧随，不阻塞 P0–P1。  
- 若保留曲线作次要/调试用，**不**在本期单独做大改版；「最近邻」曲线语义若废弃曲线则自然下线。  
- 后端出题协议变更（除非 P1 选择「提交无效标记」且仓库协议需扩展）。

## 11. 关联：可视化方向（音柱替代主时间线）

- 产品倾向：**弱化或移除**以时间为横轴的音分曲线作为主反馈；用 **目标 vs 用户** 的 **音高柱** 做直观对比；**稳/抖** 通过 **音柱高度随采样的浮动** 感知。  
- 题型与柱布局、与现有 `SightSingingPitchGraphView` 的演进顺序：见 **`docs/cursor/6c75954f/ui-ux.md`**。

## 9. 风险与回滚

- **风险**：聚合与更长窗口可能略改变「对/错」边界，需与产品确认是否同步调整文案阈值（如当前 `≥6` 为对）。  
- **回滚**：P2 开关默认关；P1 可通过 feature flag 或编译常量快速关闭聚合路径。

## 10. 自检（spec 质量）

- 无 `TBD` 占位；数值阈值明确标定为「实现前用日志标定」；**防串音间隔**为建议区间，真机标定后写入实现计划。  
- 与头脑风暴结论一致：优先判定阶段拾音，真机，无「仅示范/仅首次」规律。  
- 已纳入产品向交互：**示范后直通判定**（第 3.3 节、`P-UX`）；底栏 **示范 ∥ 下一题**、**无跳过示范** 以 **`ui-ux.md` §6** 为准。  
- 范围边界见第 8 节；整页动效大改版仍为非目标，**底栏与直通相关文案/按钮层级**允许小改。  
- 可视化主形态以 **`ui-ux.md`** 为准（音柱为主、时间线曲线弱化或移除）。
