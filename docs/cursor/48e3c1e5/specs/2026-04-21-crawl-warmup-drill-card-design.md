# 爬格子热身 · 随机题卡设计说明

> 分支：`cursor/48e3c1e5`  
> 日期：2026-04-21  
> 来源：brainstorming 已定稿（实现前规格，非代码）

## 1. 背景与问题

当前练习 Tab 中任务 `scale-walk`（展示名「爬格子热身」）进入后为通用 `PracticeTimerSessionView`：仅任务描述 + 计时 + 开始/暂停/结束，**缺少可执行的爬格子内容**，用户感知为「空壳热身」。

## 2. 目标

在 **Swift iOS Host 工程（`swift_ios_host`）** 内，将「爬格子热身」升级为：

- 每次进入练习页（`onAppear`）按当前难度 **随机生成一张爬格子题卡**；
- 同一页面内提供 **「下一组」**，在不离开页面的情况下再次随机抽题；
- 难度 **初 / 中 / 高** 在右上角 **齿轮设置** 中选择；切换难度后 **立即按新难度重抽**；
- **记住上次所选难度**（与「节奏扫弦」页一致，采用 `@AppStorage` 等机制，具体键名实现时与现有风格对齐）；
- **一期不包含内置节拍器音频**；仅展示 **建议 BPM** 文案，可与外置节拍器或音乐一起练（与扫弦页口径一致）。

非目标（本期不做）：

- 音频播放、内置节拍器 click、MIDI、自动检测弹奏是否正确；
- 与 `swift_app` 传统爬格子模块强行复用或跨工程强耦合（除非后续单独立项对齐）。

## 3. 方案选择（已定）

采用 **新建专用练习视图**（工作名：`ScaleWarmupSessionView` 或等价命名），结构与 `RhythmStrummingView` 对齐：

- `PracticeLandingView` 中 `case "scale-walk"` 路由到该新视图，不再使用 `PracticeTimerSessionView`；
- 计时、结束、保存、返回拦截等行为与现有计时练习 **保持一致**（从当前 `PracticeTimerSessionView` 迁移或抽取共享逻辑，避免能力回退）；
- 不在 `PracticeTimerSessionView` 内堆叠大量 `task.id` 分支，降低维护成本。

## 4. 用户体验规格

### 4.1 导航与布局

- **导航标题**：使用 `PracticeTask.name`（当前为「爬格子热身」）。
- **左上**：返回（与现练习页一致，含离开确认若有）。
- **右上**：齿轮 → **练习设置** Sheet（`Form`）。

### 4.2 题卡区域（主内容）

使用与练习 Tab 一致的卡片样式（如 `appCard`），展示结构化字段（实现时可微调文案，但语义需覆盖）：

| 字段 | 说明 |
|------|------|
| 弦 / 把位 | 例如「⑥～④弦」「1～4 品」等可读中文 |
| 型 / 模式 | 如半音四品型、指定弦序等，与难度池对应 |
| 建议 BPM | 正整数，仅文案 |
| 量 | 建议轮数或单次练习时长建议（与难度档一致） |
| 提示 | 一条短句：力度均匀、减少杂音等 |

**随机规则**：每次 `onAppear` 抽一组；点击「下一组」再抽一组。不要求同日固定（用户已选「每次进入重新抽」；「下一组」与之兼容）。

**去重**：生成器提供 `next(excluding:)` 或等价逻辑，避免与当前题卡 **连续重复**（与 `StrummingPatternGenerator` 思路一致）。

### 4.3 计时与结束

- 保留 **开始 / 暂停 / 结束** 与结束后的 **`PracticeFinishDialog`** 流程；
- 保存时 **`completed` / 难度星级 / 备注** 等行为与现有练习一致；
- 若当前计时页有「未开始不可结束」等提示，新页 **保持一致**。

### 4.4 练习设置 Sheet

- `Section("难度")`：`Picker` 初 / 中 / 高（枚举 rawValue 与现有练习难度中文一致即可）；
- 辅助说明：难度影响题卡池子范围；变更后立即重抽；
- 可选 `Section`：简短练习说明（爬格子热身目的、可与外置节拍器同练）。

## 5. 技术设计

### 5.1 新类型与生成器

- **`ScaleWarmupDrill`（或等价命名）**：`id: String`、展示用字段（弦范围、品位范围、模式名、建议 BPM、轮数/时长建议、提示文案等）。
- **`ScaleWarmupGenerator`**：
  - `static func nextDrill(difficulty:excluding:using:) -> ScaleWarmupDrill`；
  - 按难度维护 **非空** 题卡池；单元测试保证池非空、分档正确、连续两次不重复（在池大小大于 1 时）。

### 5.2 会话持久化（推荐）

为便于练习历史展示与对账，在 `PracticeSession` 中新增可选字段：

- **`scaleWarmupDrillId: String?`**（命名实现时可微调，须与 `rhythmPatternId` 风格一致）；
- `PracticeLocalStore` 编码 / 解码时读写该键；**旧数据缺键视为 `nil`**，不得崩溃；
- 保存会话时写入 **当前题卡的 `id`**。

若产品后续坚持不改会话模型，唯一退路为把摘要写入备注，但 **结构化差、不推荐**，本规格以新增可选字段为默认。

### 5.3 路由

- `PracticeLandingView`：`"scale-walk"` → 新视图，注入同一 `PracticeTask` 与 `PracticeSessionStore`。

### 5.4 依赖与复用

- UI 组件、主题、Finish 对话框、Store 接口与现有练习模块保持一致；
- 不引入 Flutter；不强制依赖 `swift_app`。

## 6. 测试与验收

### 6.1 单元测试（P0）

- `ScaleWarmupGenerator`：各难度至少一条；`excluding` 在池多元素时不返回同一 `id`；
- 若有 JSON 解码变更：`PracticeLocalStore` / `decodeSessions` 对缺少 `scaleWarmupDrillId` 的旧记录仍解析成功。

### 6.2 手工验收（P0）

- 进入爬格子热身：可见题卡 + 计时；再进入一次题卡内容可变；
- 切换难度：题卡立即变化且符合该档强度趋势；
- 「下一组」：题卡变化，且不明显连续重复；
- 完整保存一条记录后，历史或本地 JSON 中可看到 `scaleWarmupDrillId`（若已实现字段）；
- 无内置音频（确认无意外引入 AVAudio 等）。

## 7. 风险与后续

- **题卡音乐性**：文案型题卡无法替代老师示范；后续若加图示或简谱需另立需求。
- **与「音阶训练」推荐模块命名**：`RecommendationModuleType.scaleTraining` 标题仍为「音阶训练」，与本任务展示名不同；本期 **不强制改名**，避免范围蔓延。
