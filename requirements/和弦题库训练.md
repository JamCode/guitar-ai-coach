# 和弦题库训练（Quiz）— 产品需求

> 状态：草案（用于评审与原型联调）  
> 关联能力：复用「和弦字典」中的和弦符号、按法图、难度语义，形成“查完就练”的记忆闭环。

---

## 1. 背景与目标

### 1.1 用户痛点

- 用户知道和弦名，但记不住对应按法。  
- 仅看字典属于“被动查看”，缺少“主动回忆”的训练过程。  
- 练习后没有错题追踪，记忆容易回落。

### 1.2 产品目标

提供「和弦题库训练」：用户按难度做题，通过“看和弦名选按法”强化记忆，并获得分性质的学习反馈。

### 1.3 成功标准（方向性）

- 用户能在 1 分钟内开始一次训练并完成至少 5 题。  
- 用户可看到按“和弦性质”统计的正确率，知道自己薄弱项。  
- 用户可对错题进行二次训练，形成复习闭环。

---

## 2. 范围

### 2.1 MVP（第一版建议包含）

| 项 | 说明 |
| --- | --- |
| 训练入口 | 在字典页提供「去训练」入口；可后续扩展到主页。 |
| 难度选择 | 初级 / 中级 / 高级三级。 |
| 训练节奏 | **不限题量**，做完一题即可进入下一题；用户主动结束本次训练。 |
| 单选题型 | 题干显示和弦名，4 选 1 指法图。 |
| 立即反馈 | 提交后显示对错、正确答案、简短记忆提示。 |
| 结果页 | 显示分数、正确率、用时、错题列表。 |
| 错题再练 | 支持“一键再练错题”。 |
| 性质统计 | 按和弦性质展示正确率（如大三/小三/属七等）。 |

### 2.2 暂不纳入 MVP

- 听音题（播放后选和弦）。  
- 多选题（一个和弦多按法都算对）。  
- PvP 排行榜、好友对战。  
- 教材/歌曲驱动的个性化套题。

## 3. 难度体系（按“和弦性质”组织题库）

> 题库主轴采用“和弦性质”；同一和弦名可因按法复杂度出现在不同难度。

### 3.1 初级

- 目标：建立基础“和弦名 -> 手型”映射。  
- 性质范围：大三、小三、属七（高频）。  
- 按法特征：以开放和弦为主，尽量少横按。  
- 示例：`C` `G` `Am` `Em` `D` `A` `E` `Dm` `A7` `D7` `E7`

### 3.2 中级

- 目标：掌握常见扩展性质与部分横按。  
- 性质范围：`maj7` `m7` `sus2` `sus4` `add9` + 常见横按三和弦。  
- 按法特征：允许出现横按与中把位。  
- 示例：`Bm` `F`（标准横按）`Cmaj7` `Am7` `G7` `Dsus4` `Asus2`

### 3.3 高级

- 目标：处理复杂性质与多位置变体。  
- 性质范围：`dim` `aug` `m7b5` 以及部分转位/slash。  
- 按法特征：高把位、跨度更大、手型更复杂。  
- 示例：`C#m7b5` `Gaug` `Bbmaj7` `D/F#` `G/B` `F#m7`

### 3.4 大模型分类策略（用于离线标注）

- 输入：和弦符号、候选按法结构（品位、是否横按、跨度、开放弦数量）。  
- 输出：`difficulty`、`quality`（和弦性质）、`reason`、`confidence`。  
- 规则兜底：
  - 完整横按：最低中级。  
  - 大跨度/高把位：可提升一级。  
  - 复杂变化音：倾向高级。  
- 冲突处理：模型与规则冲突时，按更高难度入库并标注“待复核”。

## 4. 题目与组卷规则

### 4.1 题型定义（MVP）

- 题干：显示和弦符号（如 `Am7`）。  
- 选项：4 张按法图卡片，单选。  
- 判定：1 个标准答案（避免首版认知混乱）。

### 4.2 干扰项策略

- 同性质优先（例如 `m7` 题的干扰项也优先来自 `m7`）。  
- 同根音次之（如 `Am` vs `A`）。  
- 相近难度约束：避免初级题混入明显高级按法。

### 4.3 出题与练习节奏建议

- 不限制题量；每次答完点击「下一题」继续。  
- 提供「结束本次训练」入口，随时结算当前成绩。  
- 长会话中应自动覆盖多种和弦性质，避免连续只出同一性质。  
- 最近 5 题不重复同一和弦名（若题池不足再放宽）。  
- 错题权重提升：近期答错的和弦更易再次出现。

---

## 5. 数据与生成策略

### 5.1 结论：离线预生成入库为主

- 首选“离线批处理 + MySQL 入库”，在线练习直接读库。  
- 不建议“每题实时请求大模型”：成本高、延迟高、题目一致性差。

### 5.2 推荐流程

1. 从和弦字典候选池收集“和弦 + 多按法”。  
2. 离线调用大模型做性质/难度标注。  
3. 规则引擎校验并生成干扰项候选。  
4. 写入 MySQL 题库。  
5. 在线按难度与用户历史进行组卷。

### 5.3 在线兜底（非 MVP，可后续）

- 若某难度题量不足，可临时生成小批题并进入“待审核池”。  
- 审核通过后再写入正式题库，保证长期稳定性。

### 5.4 学习者身份（MVP 采用方案 B）

- 当前无账号体系，采用匿名 `learner_id` 记录训练数据。  
- `learner_id` 由前端首次生成并本地持久化，请求时随训练接口上报。  
- 后续接入登录后，再通过映射表将匿名记录合并到正式账号。

---

## 6. 用户流程（MVP）

1. 进入训练页，选择难度（初/中/高）。  
2. 开始答题：逐题选择按法并提交。  
3. 每题即时反馈：对错 + 记忆提示；点击「下一题」继续。  
4. 用户主动点击「结束本次训练」，查看结果（总分、正确率、性质维度表现、错题）。  
5. 可选择「继续训练」或「再练错题」。

---

## 7. 验收标准（MVP 勾选用）

- [ ] 用户可在 2 步内开始答题（进入训练 -> 选难度 -> 开始）。  
- [ ] 初/中/高均支持连续出题（不限题量，题池可循环抽取）。  
- [ ] 最近 5 题不出现重复和弦名（题池数量不足时有可解释兜底）。  
- [ ] 每题均有且仅有 1 个标准答案。  
- [ ] 提交后立即反馈对错，并可看到正确按法。  
- [ ] 结果页含总分、正确率、错题列表、按性质统计。  
- [ ] 用户可随时结束并结算；错题再练可正常作答。  
- [ ] 训练过程无白屏，异常时有可理解提示。
