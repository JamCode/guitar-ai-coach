# 和弦题库训练 — 数据结构（方案 B：匿名 learner_id）

> 目标：在无账号体系阶段，支持连续训练、错题再练、结果统计与后续账号迁移。

---

## 1. 身份策略

- 不使用 `user_id`，统一使用 `learner_id`（匿名设备 ID）。  
- 前端首次进入训练页时生成 UUID（如 `anon_xxx`），持久化到 `localStorage`。  
- 每次训练请求带上 `learner_id`。  
- 若用户清缓存，视为新学习者（MVP 可接受）。

---

## 2. MySQL 表设计（MVP）

## 2.1 题目主表 `quiz_question`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT PK | 题目 ID |
| `difficulty` | VARCHAR(16) | `beginner/intermediate/advanced` |
| `chord_symbol` | VARCHAR(32) | 和弦名，如 `Am7` |
| `chord_quality` | VARCHAR(32) | 和弦性质，如 `m7` |
| `correct_fingering` | VARCHAR(32) | 标准按法编码 |
| `status` | VARCHAR(16) | `draft/active/offline` |
| `version` | VARCHAR(32) | 题库版本，如 `v1_2026_03` |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

建议索引：
- `idx_question_difficulty_status`(`difficulty`, `status`)
- `idx_question_version`(`version`)
- `idx_question_symbol`(`chord_symbol`)

## 2.2 题目选项表 `quiz_question_option`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT PK | 选项 ID |
| `question_id` | BIGINT | 关联 `quiz_question.id` |
| `option_key` | CHAR(1) | `A/B/C/D` |
| `fingering` | VARCHAR(32) | 选项按法编码 |
| `is_correct` | TINYINT(1) | 是否正确答案 |
| `display_order` | INT | 展示顺序 |
| `created_at` | DATETIME | 创建时间 |

建议索引：
- `idx_option_question`(`question_id`)
- `uniq_question_option_key`(`question_id`, `option_key`) UNIQUE

## 2.3 训练会话表 `quiz_attempt`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT PK | 会话 ID |
| `learner_id` | VARCHAR(64) | 匿名学习者 ID |
| `difficulty` | VARCHAR(16) | 当前难度 |
| `started_at` | DATETIME | 开始时间 |
| `ended_at` | DATETIME NULL | 结束时间 |
| `total_answered` | INT | 本次作答题数 |
| `total_correct` | INT | 本次答对题数 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

建议索引：
- `idx_attempt_learner`(`learner_id`, `started_at`)
- `idx_attempt_difficulty`(`difficulty`, `started_at`)

## 2.4 作答明细表 `quiz_attempt_answer`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT PK | 明细 ID |
| `attempt_id` | BIGINT | 关联 `quiz_attempt.id` |
| `question_id` | BIGINT | 关联 `quiz_question.id` |
| `selected_option_key` | CHAR(1) | 用户选择 |
| `is_correct` | TINYINT(1) | 是否答对 |
| `answered_at` | DATETIME | 作答时间 |

建议索引：
- `idx_answer_attempt`(`attempt_id`)
- `idx_answer_question`(`question_id`)

---

## 3. 初始化入库流程

1. 离线生成题库（和弦 + 正确按法 + 3 个干扰项 + 难度 + 性质）。  
2. 批量导入 `quiz_question`。  
3. 按 `question_id` 批量导入 `quiz_question_option`。  
4. 设置 `status=active`、`version=v1_2026_03`。  
5. 上线查询仅拉取 `status='active'` 的题目。

---

## 4. 与“最近 5 题不重复”的关系

- 在线出题时按 `learner_id` 读取最近作答记录（可用内存+落库双策略）。  
- 先排除最近 5 题同 `chord_symbol`，若候选不足再放宽。  
- 该规则建议在服务层实现，前端只负责展示。

---

## 5. 后续迁移到账号体系

- 新增 `user_id` 后，不改现有逻辑，先保留 `learner_id`。  
- 增加映射表：`learner_identity_map(learner_id, user_id, linked_at)`。  
- 登录后把匿名历史训练合并到用户维度统计。  
- 稳定后可逐步将 `quiz_attempt` 的主查询维度由 `learner_id` 切换到 `user_id`。

---

## 6. 非目标（当前不做）

- 多端实时同步（同一账号跨设备）。  
- 复杂反作弊与排行榜。  
- 全量题库在线实时生成。
